## 1. システム概要

院内の職員給食（朝食・昼食・夕食・夕食医師）の注文を管理するシステムです。
院内ネットワーク上のどの端末のブラウザからでも同じデータにアクセスできます。

### 対応ブラウザ
- Google Chrome
- Microsoft Edge

### 食事の種類

**職員給食（4種）**
| 食事 | 説明 |
|------|------|
| 朝食 | 全職員共通 |
| 昼食 | 全職員共通 |
| 夕食 | 全職員共通 |
| 夕食医師 | 医師用夕食 |

**検査食（3種）**
| 食事 | 説明 |
|------|------|
| 検査朝・検査昼・検査夕 | 医師が食事チェックのために食べる検査食。個人が注文するのではなく、管理者が「検査食管理」で日ごとに担当医師を割り当てる |

**保育園食（5種）**
| 食事 | 説明 |
|------|------|
| 朝食 | 普通食 / きざみ食 |
| 10時おやつ | 普通食 / きざみ食 |
| 昼食 | 普通食 / きざみ食 |
| 15時おやつ | 普通食 / きざみ食 |
| 夕食 | 普通食 / きざみ食 |

## 2. アクセス方法

ブラウザのアドレスバーに以下を入力してください。

- **職員給食**: `http://<サーバーのIPアドレス>/index.html`
- **保育園食**: `http://<サーバーのIPアドレス>/hoikuen/index.html`

画面右上の「保育園食へ」「職員給食へ」リンクで相互に移動できます。

## 3. 一般機能

一般の職員がパスワードなしで使用できる機能です。

### 3.1 本日の給食（トップページ）

その日の給食注文者を朝食・昼食・夕食・夕食医師ごとに一覧表示します。

- **日付切替**: 「<」「>」ボタンまたは日付欄で日付を変更
- **今日ボタン**: 今日の日付に戻る
- 土日・祝日の場合は画面上部に通知が表示されます

### 3.2 注文入力

職員の月間給食注文を入力します。

#### 基本操作
1. **年月を選択** — 注文したい月を選びます
2. **部署を選択**（任意） — 部署で絞り込めます
3. **職員を選択** — 注文する職員を選びます
4. カレンダーが表示されるので、各日の **朝食・昼食・夕食・夕食医師** のチェックボックスをオン/オフします
5. チェックを入れた時点で自動的にサーバーに保存されます
6. **「確定」ボタン** を押すと注文が確定されます

#### 一括操作ボタン
| ボタン | 動作 |
|--------|------|
| 平日朝食 | 月～金（祝日除く）の朝食を一括チェック |
| 平日昼食 | 月～金（祝日除く）の昼食を一括チェック |
| 平日夕食 | 月～金（祝日除く）の夕食を一括チェック |
| 平日夕医師 | 月～金（祝日除く）の夕食医師を一括チェック |
| 前月コピー | 前月の注文内容をそのままコピー |
| 全クリア | 選択中の月の全注文を削除 |

#### 確定と修正
- **未確定** の注文はいつでも変更できます
- **「確定」** を押すとロックされ、チェックボックスが操作できなくなります
- 確定済みの注文を変更するには **「修正」ボタン** を押します
  - 管理者パスワードが設定されている場合、パスワード入力が必要です

#### 職員の切替
- 「< 前」「次 >」ボタンで前後の職員に切り替えられます

### 3.3 入力履歴

注文の変更履歴を確認できます。

- **日付** を指定して、その日に行われた操作を表示します（初期値は当日）
- **職員** で絞り込みができます
- **操作種別** で絞り込みができます（変更・確定・修正開始など）

### 3.4 カレンダーの色分け

| 色 | 意味 |
|----|------|
| 青色 | 土曜日 |
| 赤色 | 日曜日 |
| 黄色 | 祝日 |
| 緑の左線 | 今日 |

## 4. 管理者機能

管理者パスワードを入力してアクセスする機能です。

### 4.1 管理者モードへの切替

1. 画面上部の **「管理者」ボタン** をクリック
2. パスワードを入力（初期パスワード: **jinji**）
3. 管理者専用タブ（集計・職員マスタ・休日マスタ・データ管理）が表示されます
4. 解除するには **「管理者モード解除」ボタン** をクリック

### 4.2 集計（管理者専用）

月次の注文集計を行います。

#### 集計実行
年月を選んで **「集計実行」** をクリックすると、以下が表示されます。
- **月次合計**: 朝食・昼食・夕食・夕食医師の合計食数
- **部署別集計**: 部署ごとの食数内訳
- **日別内訳**: 日ごとの食数

#### 全注文者一覧
**「全注文者一覧」** をクリックすると、注文のある全職員の日別注文一覧表を表示します。
- 各職員の各日について朝・昼・夕・医師の注文を ○ で表示
- 土日祝は色分け表示
- 右端に合計列

#### CSV出力
**「CSV出力」** をクリックすると、全職員の月間注文数をCSVファイルでダウンロードします。
- 形式: 職員ID, 氏名, 部署, 朝食, 昼食, 夕食, 夕食医師, 合計
- Excel対応（UTF-8 BOM付き）

#### 食数表Excel出力
- **「総務課提出用Excel」**: 「職員食実施食数表」を出力。列=1〜31日+合計、
  行=朝・昼・夕それぞれ医局/一般/合計。
  医局は部署名に「医局」を含む職員（夕の医局は「夕食医師」のチェック数）
- **「栄養科掲示用Excel」**: 「職員食食数表」を出力。1〜15日と16〜31日の2ブロック横並び、
  列=日/朝/昼/夕/医局。昼は昼合計+1（検食分）

#### 印刷
**「印刷」** をクリックすると、表示中の集計結果を印刷します。
- 全注文者一覧は横向き（landscape）で印刷されます

### 4.3 職員マスタ（管理者専用）

職員情報の登録・編集・削除を行います。

#### 個別登録
1. 職員ID・氏名・部署を入力
2. **「登録」** をクリック

#### 編集
1. 職員一覧の **「編集」** をクリック
2. フォームに情報が入るので修正
3. **「更新」** をクリック

#### 削除
- 職員一覧の **「削除」** をクリック（確認ダイアログあり）
- 注文データも一緒に削除されます

#### CSV一括取込
1. **「ファイルを選択」** でCSVファイルを選択
   - 形式: `職員ID,氏名,部署`（UTF-8）
   - ヘッダ行あり/なし両対応
2. 「既存IDを上書き」にチェックを入れると、既存職員の情報を更新
3. **「取込実行」** をクリック

#### CSV出力
- **「CSV出力」** で現在の職員一覧をCSVダウンロード

#### 検索
- 検索欄にID・氏名・部署を入力すると絞り込み表示

#### 子供管理（保育園）
保育園食の注文に必要な子供の登録を行います。
1. 職員検索欄でID・氏名・部署を入力して保護者を検索
2. 職員を選択
3. 子供の名前を入力して **「追加」**
4. 一覧から **「削除」** で削除

### 4.4 休日マスタ（管理者専用）

祝日・特別休日の管理を行います。注文入力の「平日○○」ボタンで除外される日を設定します。

#### 個別登録
1. 日付と名称を入力
2. **「登録」** をクリック

#### 祝日データ初期化
- **「祝日データ初期化（2026-2028年）」** をクリックすると、2026～2028年の日本の祝日を一括登録
- 既存データに追加されます（上書きではない）

#### 表示・削除
- 表示年を切り替えて祝日一覧を確認
- **「削除」** で個別に削除

### 4.5 データ管理（管理者専用）

#### 編集パスワード設定
確定済み注文の修正や管理者モードに必要なパスワードを設定します。
- **「パスワード設定」**: 新しいパスワードを設定
- **「パスワード解除」**: パスワードを削除（誰でも管理者モードに入れるようになります）

#### データエクスポート
全データ（職員・注文・休日・履歴等）をJSONファイルとして保存します。
- 定期的なバックアップとして推奨

#### データインポート
エクスポートしたJSONファイルからデータを復元します。
- **既存データは上書きされます** — 事前にエクスポートしてバックアップを取ってください

#### データ初期化
全データを削除します。
- **この操作は元に戻せません**

### 4.6 履歴クリア（管理者専用）

入力履歴タブの **「履歴クリア」ボタン** は管理者モード時のみ表示されます。
- パスワード入力が必要です
- 全履歴を削除します

### 4.7 検査食管理（管理者専用）

各日の朝・昼・夕の検査食を食べる医師を割り当てます。

#### 割り当て方法
1. 管理者モードで **「検査食管理」タブ** を開く
2. 年月を選択
3. カレンダーの各日の検査朝・検査昼・検査夕のプルダウンから医師を選択
   （プルダウンには診療部の職員のみ表示されます）
4. 最後に **「登録」ボタン** を押して保存します
   （保存に失敗した場合はエラーが表示されます）

#### 医師別集計
- 検査食管理タブの下部に、表示中の月の医師別割り当て回数（検査朝・検査昼・検査夕・合計）が表示されます
- 集計タブの「集計実行」でも **医師別 検査食統計** が表示されます
- 集計タブの **「検査食統計CSV」** ボタンで医師別統計をCSV出力できます（列: 職員ID, 氏名, 部署, 検査朝, 検査昼, 検査夕, 合計）

#### 本日の給食への表示
- 本日の給食タブに、その日の検査食担当医師（朝・昼・夕）が表示されます

### 4.8 食事数合計（注文＋検査食）

実際に用意する食事数は、注文（朝・昼・夕・夕食医師）に検査食を加えた数になります。
以下の場所で「注文」「検査食」「合計」を分けて確認できます。

- **本日の給食タブ**: その日の朝の合計・昼の合計・夕の合計・夕食医師の合計・検査食の合計
- **集計タブ**: 「集計実行」で月次の食事数合計を表示
- **CSV出力**: 月間注文集計CSVの先頭に食事数合計を出力
- **全注文者一覧Excel出力**: 表の先頭に食事数合計を出力

※「検査食の合計」は朝・昼・夕の合計にすでに含まれる内数です。
「食事総数」は 朝の合計＋昼の合計＋夕の合計＋夕食医師の合計 です。

## 5. 保育園食注文管理

院内保育園に預けている子供の食事注文を管理します。

### 5.1 事前準備
1. 職員給食の **職員マスタ** で保護者（職員）を登録
2. **子供管理** で職員に子供を紐づけて登録

### 5.2 本日の食事

当日の保育園食注文者を朝食・10時おやつ・昼食・15時おやつ・夕食ごとに表示します。
- 子供名、保護者名、食種（普通食/きざみ食）を一覧表示

### 5.3 注文入力

1. 年月・保護者・子供を選択
2. カレンダーのセルをクリックして食種を切替
   - **クリック1回目**: 普（普通食・青色）
   - **クリック2回目**: き（きざみ食・オレンジ色）
   - **クリック3回目**: 取消（空欄に戻る）
3. 変更は即時保存されます

#### 一括操作
| ボタン | 動作 |
|--------|------|
| 平日全食セット | 月～金（祝日除く）の全食を普通食でセット |
| 前月コピー | 前月の注文内容をコピー |
| 全クリア | 全注文を削除 |

### 5.4 集計

月次の注文集計（合計・保護者別・日別）を表示・印刷できます。

### 5.5 入力履歴

保育園食の入力履歴を月・保護者で絞り込んで確認できます。

## 6. サーバー管理者向け

### 6.1 動作環境
- **サーバー**: Windows 11 + Apache 2.4 + PHP
- **通信**: 院内LAN（インターネット不要）

### 6.2 ファイル構成

```
ドキュメントルート/
├── index.html        ... 職員給食トップページ
├── app.js            ... 職員給食ロジック
├── style.css         ... 職員給食スタイル
├── api.php           ... データAPI（サーバー側処理）
├── data/             ... データ保存フォルダ（自動生成）
│   ├── staff.json        ... 職員マスタ
│   ├── orders.json       ... 注文データ
│   ├── holidays.json     ... 休日マスタ
│   ├── history.json      ... 入力履歴
│   ├── children.json     ... 子供データ
│   ├── config.json       ... 設定（パスワード等）
│   ├── confirmed.json    ... 確定状態
│   ├── kensa.json        ... 検査食の医師割り当て
│   ├── hoiku_orders.json ... 保育園注文データ
│   ├── hoiku_history.json... 保育園入力履歴
│   └── hoiku_confirmed.json ... 保育園確定状態
└── hoikuen/
    ├── index.html    ... 保育園食トップページ
    ├── app.js        ... 保育園食ロジック
    └── style.css     ... 保育園食スタイル
```

### 6.3 アップデート時の注意

プログラムを更新する際は、以下のファイルだけを上書きしてください。

**上書きするファイル:**
- `api.php`
- `app.js`
- `index.html`
- `style.css`
- `hoikuen/app.js`
- `hoikuen/index.html`
- `hoikuen/style.css`

**触らないファイル（データが消えます）:**
- `data/` フォルダ内のすべてのファイル

### 6.4 バックアップ

- **推奨**: 管理者モード → データ管理 → エクスポートで定期的にJSONバックアップ
- **サーバー側**: `data/` フォルダを丸ごとコピーしてバックアップ

### 6.5 トラブルシューティング

| 症状 | 対処 |
|------|------|
| データが保存されない | `data/` フォルダのアクセス権限を確認。PHPが書き込み可能か。 |
| 他の端末にデータが反映されない | 全端末が同じサーバーのURLにアクセスしているか確認。 |
| ボタンが反応しない・新機能が見えない | 古いapp.jsがキャッシュされている。**Ctrl+F5**（スーパーリロード）を実行。 |
| 祝日が反映されない | 管理者モード → 休日マスタ → 祝日データ初期化を実行。 |

### プログラム更新時の注意（キャッシュ対策）
`index.html` 内の `app.js?v=20260716` の `v=` の値は更新日ごとに変えてあります。
サーバーのファイルを更新すると、各PCのブラウザは次回アクセス時に自動で新しい
JavaScriptを読み込みます（Ctrl+F5不要）。手動でファイルを編集した場合は
`v=` の日付も変更してください。
